About MicroQuest

A tiny gamified dashboard built as an individual coding project.

Project Goals

  • Create a completely new website with original content and styling.
  • Practice working with JavaScript data structures and localStorage.
  • Integrate at least one external JavaScript plugin (Chart.js, SortableJS).
  • Experiment with a “gamified” interface to track tasks and habits.

Key Features

  • Dashboard: level, XP progress bar, and a 7-day completion chart.
  • Quest Board: add quests, move them between columns, and drag cards with SortableJS.
  • Habits & Streaks: predefined habits, daily check-ins, streak calculation, and a weekly bar chart.
  • Local Storage: all quests and habits are stored in the browser, no server required.

Technical Stack

  • HTML5 semantic structure and multi-page navigation.
  • Custom CSS with a dark, “HUD-like” interface and responsive layout.
  • Vanilla JavaScript modules for data modeling, filtering, and UI updates.
  • Chart.js for data visualization and SortableJS for drag-and-drop interactions.

Reflection

While building MicroQuest, I learned how to design my own data structures, connect UI components to localStorage, debug plugin integration, and think about user experience beyond static pages. The most challenging part was keeping the code organized while supporting multiple pages, but it helped me understand how a slightly more complex front-end app is structured.

Built with HTML, CSS, JavaScript, Chart.js, and SortableJS.
